    - Create new admin accounts.

### Backup

- Go to **System → Maintenance → Backup & Restore** (or equivalent). [Fortinet Documentation+2Fortinet Help Center+2](https://docs.fortinet.com/document/fortigate/7.6.4/administration-guide/702257/configuration-backups-and-reset?utm_source=chatgpt.com)
    
- Choose to save the current configuration. You may have options:
    
    - Plaintext (unencrypted) backup.
        
    - Encrypted backup (you set a password). [Scribd+1](https://www.scribd.com/document/734358285/Exercise-3?utm_source=chatgpt.com)
        
- Store the backup securely (especially if encrypted: you must remember the password).
    

### Restore

- In that same menu choose “Restore”, upload the file.
    
- If the backup is encrypted, you’ll be prompted for the password. [Scribd+1](https://www.scribd.com/document/734358285/Exercise-3?utm_source=chatgpt.com)
    
- After restore, device may reboot.
    
- **Important**: Ensure firmware version matches (or is compatible) with what the backup was taken on. Mismatch can cause missing elements (e.g., admin account gone). [Omnimitra Teknologi Solusindo](https://omnimitra.id/en/blog/tips-%26-trick/fortigate-restore-configuration-issue%2C-but-missing-admin-account%2C-only-normal-user-exists?utm_source=chatgpt.com)
    

### Revisions / Versioning

- Another useful feature: “Configuration Revisions”. This allows you to revert to previous config states internally. [Fortinet Community](https://community.fortinet.com/t5/FortiGate/Technical-Tip-How-to-save-and-restore-configuration-changes/ta-p/195879?utm_source=chatgpt.com)
    
- Typically found in GUI under something like “Configuration → Revisions” (top-right dropdown) — you can view and revert past configs.

## 4. Practical Tips for Your Lab Setup

- Since you’re building a lab (2 FortiGates, Ubuntu VMs, VLANs, site-to-site IPsec), **take backups often** before big steps (HA config, IPsec).
    
- Label your backup files clearly (e.g., “FGT-HA-before-IPsec-20251030.conf”) to know what you restored.
    
- When you restore a config on a different device model or firmware version, **check all admin accounts** are present and working (see article about missing admin account after restore). [Omnimitra Teknologi Solusindo](https://omnimitra.id/en/blog/tips-%26-trick/fortigate-restore-configuration-issue%2C-but-missing-admin-account%2C-only-normal-user-exists?utm_source=chatgpt.com)
    
- Use admin profiles: For example, create one admin with full rights for major changes, another with limited rights (monitoring only) — good for auditing and security.
    
- Use configuration revisions for quick rollback if a change breaks connectivity (especially relevant in site-to-site VPNs).
